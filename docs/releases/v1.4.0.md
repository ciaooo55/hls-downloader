# HLS Downloader v1.4.0

这是一次桌面端、浏览器接管、请求身份和下载稳定性的联合升级。重点不是增加一个孤立按钮，而是把“用户点击 → 浏览器识别 → 桌面确认 → 按原请求身份下载 → 失败后给出可操作原因”这条链路完整收紧。

## 桌面端与界面

- 新增 Kotlin/Compose Desktop 原生主程序，安装包和便携包自带 Java 运行时，不再依赖 WebView2 才能显示主界面。
- 桌面布局、侧栏、任务区、设置分组、明暗主题和视觉层级按 Cockpit Tools 的简洁控制台风格重新整理；同时保留 React 前端用于播放器和本地服务回退页面。
- 新增原生的新建任务、批量添加、浏览器接管确认、任务日志、设置、文件夹选择和系统托盘交互。
- 改进单实例激活、桌面端与下载核心的启动/退出握手，以及安装覆盖升级时的旧进程清理。
- 更新桌面端、安装程序、前端和 Chrome/Firefox 扩展图标，并补齐不同尺寸资源。

## 浏览器接管与识别

- 按发布要求移除旧油猴脚本、后端脚本接口、桌面脚本入口和脚本产物；v1.4.0 只提供受测的 Chrome/Edge 与 Firefox 插件。
- Chrome 和 Firefox 扩展升级到 v1.4.0；严格区分“用户明确下载点击”和普通播放、展开、登录等页面交互。
- 支持真实链接、下载语义按钮、打开新标签页的下载链、右键下载、magnet，以及未知 Content-Length 的下载；已知的小文件仍按阈值留给浏览器。
- 资源按“标签页 + 当前页面 URL”隔离，页面跳转会清除旧徽标和过期资源，不再把其他标签页或前一页面的视频混入当前面板。
- HLS 多清晰度结果、可见视频右上角下载入口、折叠资源面板和建议文件名均做了细节修正。
- 桌面接管设置会与扩展同步。桌面端离线、确认传输失败或未真正接收任务时，浏览器下载会恢复，避免文件无声丢失。
- 扩展测试现在强制执行 TypeScript 严格检查，Chrome MV3 与 Firefox MV3 使用同一套受测识别规则。

## 403、登录态与请求身份

- 不再凭空生成 `Origin`。扩展只重放浏览器真实捕获到的 Origin；DOM/Performance 识别可使用当前页面作为 Referer，但不会伪造跨站 Origin。
- 捕获并过滤可安全重放的请求头，包括 Authorization、Sec-CH-UA 和站点自定义令牌；Host、Content-Length、Range、Accept-Encoding 等由网络栈管理的字段不会被重放。
- HLS 主清单、变体清单、分片、初始化段和 AES-128 密钥按各自 URL 选择来源上下文。最多保存最近 12 个来源，Cookie/Authorization 不会从主站泄漏到未捕获的 CDN。
- 浏览器任务不再继承设置中与当前站点无关的全局 Referer、Origin 或 Cookie；手动和批量任务仍可按需使用全局请求头。
- Cookie、Authorization 和分来源请求上下文在 SQLite 中加密保存，并包含旧数据库自动迁移。
- HLS 网络层会根据捕获的 User-Agent 选择 Chrome、Firefox 或 Safari TLS 指纹，避免“Firefox 请求头 + Chrome TLS”一类明显不一致。
- 401/403 不再机械重试五次。错误详情会区分“缺少网页上下文”和“签名链接/登录会话已过期”，指导用户回到原页面刷新并重新发送。

## 下载核心稳定性

- HTTP 文件先用一字节 Range 实测服务器能力，不再只相信 HEAD/Accept-Ranges；不支持分段时自动退回单连接。
- 改进重定向后的文件名、Content-Disposition 命名、通用媒体名替换和未知大小文件处理。
- HLS、DASH、HTTP 和 BitTorrent 统一使用经过清理的任务请求身份与错误诊断；401/403 等永久性错误停止无意义重试，408/429/5xx 等瞬时错误继续按策略重连。
- 下载完成、失败、暂停、跨盘发布和程序关闭时的过程文件处理更加明确；缓存与过程目录可单独配置。
- 新增本地核心健康检查、原生桌面 API、浏览器接管状态查询和 Native Messaging 设置同步。

本版发布前通过 177 项 Python 测试、40 项 React/TypeScript 前端测试、31 项插件测试与严格 TypeScript 检查，并执行 Compose Desktop 测试、Chrome/Firefox 生产构建、Firefox `web-ext` lint、Windows 安装版/便携版打包及启动冒烟验证。所有发布文件均列入 `SHA256SUMS.txt`。

## 已知边界

- 403 不是单一问题。已经解决错误 Origin、跨标签页身份串用、未重放授权头、TLS 指纹不匹配和跨 CDN 凭据泄漏；但已过期签名 URL、已退出登录、验证码/设备绑定、地区限制和站点特定反机器人策略仍需要回到原网页刷新会话。
- DRM（Widevine/FairPlay/PlayReady）内容不提供解密或绕过功能；独立外部 HLS 音轨等尚未实现的媒体组合会明确报告不支持。
- 首次发布的安装程序没有商业代码签名证书，Windows SmartScreen 可能显示未知发布者。请只从本仓库 Release 下载并核对 SHA256。

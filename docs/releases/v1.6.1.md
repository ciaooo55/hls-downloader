# HLS Downloader v1.6.1

## 自动更新更可靠

- GitHub 匿名 API 被限流、TLS 连接中断或临时网络异常时，更新界面现在显示简短且可操作的中文提示，不再暴露 HTTP、SSL 或 urllib 内部错误。
- 已成功取得并校验过版本与 SHA-256 的情况下，即使随后重新检查受限，也可以继续安全地下载并安装该版本；不会因为第二次检查失败阻断升级。
- 更新检查失败时保留已验证的版本信息，并明确提示用户可直接安装、稍后重试，或打开 Release 页面手动下载。
- 未获得可信更新信息时，提供重新检查和打开 Release 页入口；安装包仍必须通过 SHA-256 校验后才能启动。

## 视频识别更克制

- 网页识别现在同时识别 HLS (`.m3u8`) 与 MPEG-DASH (`.mpd`)；直接 DASH 清单会作为自适应媒体资源加入下载，而非误判为普通 XML 页面。
- 对无扩展名但响应 MIME 为 HLS/DASH 的浏览器捕获资源，桌面端会正确创建 HLS/DASH 任务，不会把清单当作普通 HTTP 文件保存。
- 插件会在进入面板前过滤独立音轨、字幕、缩略图、trickplay、广告和分片清单，保留主播放清单与实际媒体资源。
- 当网页播放器是 blob/MSE、无法把网络资源和当前播放器精确对应时，“下载视频”只会打开候选面板，不再把页面上碰巧唯一的资源直接误发。

## 文件与 BT

- 普通 `.torrent` 直链现在可在浏览器资源识别中出现；上传种子会先验证，HTML 登录页或损坏文件会立即给出中文错误，不再创建一个必然失败的后台任务。
- BT 下载中可以调整文件选择；未选文件停止请求，已下载数据保留。
